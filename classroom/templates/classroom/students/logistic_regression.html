Background

Not all regression algorithms are purely used for regression problems. Some regression can be used for classification. One such example is Logistic Regression or, Logit Regression.

Logistic regression is typically used to estimate the probability of someone belonging to a particular class. For example we can estimate the possibility of a child being born prematurely and use this probability to classify a future birth as either likely to be premature or not. The logistic or logit is a sigmoid function and the probability returned would be a value between 0 and 1.

As you can tell from the above example, logistic regression is typically used to predict a binary outcome given a set of independent variables.

What is the logistic function?


Ok, let’s try and explain that again. So a regression takes some data and figures out the equation of the function that describes them. In other words you start with some data and then pick a line or shape/curve that you think will best fit your data. With linear regression this would be a straight line.

However, when it comes to a classification problem where we are trying to predict a binary outcome. Specifically, we are interested in the probability of an outcome occurring. This probability must be between 0 and 1. Using linear regression could result in a scenario where the prediction value is lower than 0 or greater than 1.

Let's look at a practical example


Let’s use this example from GitHub:

Let us assume we create a perfectly balanced dataset which contains a list of customers and a label to determine whether a customer has made a purchase. Let us assume these are customers purchasing fish tanks. In this hypothetical dataset there are 20 customers. 10 of these customers are aged between 10 and 19 and have purchased a fish tank. 10 customers aged between 20 and 29 have not purchased a fish tank.

In linear regression we would try to find the relationship between our input variables and the outcome we would like to predict. Let us assume we train a regression model based on this dataset and end up with this best fit line.

We can easily picture a scenario where we would use this model for predictive purposes. If for example the value of the Y-axis is greater than 0.5 we would predict that the particular customers in this space will likely purchase a fish tank.

The problem however is that for customers aged 10 or 30 our predicted Y-value would be greater than 1 and less than 0 respectively. Enter logistic regression.

With logistic regression we use a function that transforms the line into an s-shaped curve. This is known as the logistic function.

Now, all our data points are between the values of 0 and 1.

Let’s look at the logistic sigmoid function a little more to get a more intuitive understanding of this classification algorithm:


The logistic sigmoid function is calculated as 1 divided by 1 plus ‘e’ raised to the power of negative ‘x’. In this instance x is the numerical value you wish to transform and e represents Euler’s constant, 2.718.


The logit function is not the only function that yields S-shaped curves.

Based on the probabilities we find we can classify each datapoint to one of either binary classes we are trying to predict for. Data points that have a result of exactly 0.5 are unclassifiable but these instances are very rare.

Why we use the logit function

The logit function is not the only function that yields S–shaped curves, and it would seem that there is no reason to prefer the logit to other possible choices. For example, another function that generates S–shaped curves is the cumulative distribution function for the normal distribution; analysis using that function is called probit analysis.

Logistic regression (that is, use of the logit function) has several advantages over other methods, however. The logit function is what is called the canonical link function, which means that parameter estimates under logistic regression are fully efficient, and tests on those parameters are better behaved for small samples
