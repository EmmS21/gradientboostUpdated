Background

K-Nearest Neighbors is one of the most basic classification algorithms in Machine Learning that makes no underlying assumptions about the distribution of data. As the name implies K Nearest Neighbors makes predictions through a majority vote using the most similar K instances/ neighbors of a given data point.

We refer to K-NN as a non-parametric algorithm, meaning that it makes no assumptions about the data. It also does not learn from the training set. This means that there is no explicit training phase before the classification process. We therefore start making classifications immediately once we have our data. Other non -parametric algorithms include Decision Trees and Support Vector Machines.

The downside to this is that we need to keep our entire training set in memory making k-NN more suited for smaller datasets that do not have too many features.

Use cases


Can you picture a scenario where k-NN might be an ideal algorithm for a Machine Learning problem. Think about shopping on Amazon, when you like a particular product you will be recommended with other similar items based on your selection. The k-NN algorithm in this instance would compare the set of users who like each item and then proceed to look for similarity.

In order to determine which of the K instances are most similar in a dataset a distance measure is used. This could either be the Euclidean Distance (the most popular), Hamming Distance, Manhattan Distance, Tanimoto Distance, Jaccard Distamce, Mahalanbobis distance, Cosine distance and Minkowski Distance.

Since Euclidean distance is the most commonly used distance metric in k-NN let us define the Euclidean distance further.

Euclidean Distance
