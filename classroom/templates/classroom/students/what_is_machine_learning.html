What is Machine Learning

Sourced from Hands on Machine Learning with Scikit-Learn & TensorFlow Adapted for your convenience.

Data has been called the oil of the 21st Century as it has become pivotal in decision making, understanding both the world around us and user behaviour in the business context. According to Forbes we currently produce over 2.5 quintillion bytes of data are created each day and the amount of data we are producing is growing exponentially. However, the data itself is not where the real value lies, the value lies in insights that can be derived from the data. We can think of Machine Learning as both the science and the art of programming computers to pull relevant information from data and learn to do a task or perform a prediction on its own.

To put it in practical terms, let us assume you would like to create a program that will take in input in the form of a picture and then classify the picture as either a cat or a dog. With each picture the program gains new insights to better understand how to classify a picture as a cat or a dog, attaching weights to differentiate attributes depending on the model it uses to learn to classify either animal. The more data you have, the better the model becomes at learning a task

Let us look at a practical ‘recipe book’ of how we would create this classification program:

We would start off by collecting our data. The collection method will vary depending on what sources we are using. After this process we would then proceed to split the data into two sets; training and test data. We use the training data to train our model and the test data to test its performance.

The model we choose then looks for patterns to better understand how to differentiate between the two images.

Lastly, proceed to evaluate the accuracy of the model and either analyse its errors or launch it

Machine Learning use cases

Machine Learning has numerous practical use cases ranging from marketing insights to facial recognition. Let us look at a few use cases:

  1. Voice Assistants and other Conversational Applications (e.g Siri and Alexa)
  2. Smartphone cameras
  3. Movie, music and app recommendations
  4. Dynamic Pricing (e.g surge pricing in Uber)
  5. Optimal map routes (e.g Waze)
  6. Multichannel Marketing Attribution
  7. Self driving cars
  8. Personalized Banking
  9. Personalized Marketing

As you may have noticed from the previous page, the use cases are numerous and span across different industries. As more companies continue to embrace machine learning and data analysis in making decisions, job opportunities have continued to grow in not just data science but data engineering. You can think of data engineering as the people who create the pipelines and tools to enable data scientists and analyse to more easily retrieve cleaner data and use it to create models and derive insights to help solve business problems and guide decision making.

Identifying good problems for ML

Supervised versus Unsupervised Learning

Machine Learning models can be classified by the degree and type of supervision required during the model training process. Machine Learning models can be classified under four major categories. These are:

  1. Supervised Learning
  2. Unsupervised Learning
  3. Semi-supervised Learning
  4. Reinforcement Learning

Batch versus Online Learning

Machine Learning systems can also be classified according to whether they learn incrementally from a stream of incoming data or not.

Batch Learning

With batch learning, the system is capable of learning incrementally meaning it must be trained using all the available data. This generally takes a lot of time and computing resources, as a result it is typically carried out offline. The system is first trained, launched into production and continues to run without requiring further learning. When you need your batch learning system to gain more insights from new data you would need to train a new version from scratch on the full dataset and replace the old with the new system.

Online Learning

With online learning you instead feed the system data sequentially either individually or in small groups. Each step in the learning process is faster and cheaper in terms of computational resources allowing your system to gain insights from new data as it arrives. This is great for systems that are expected to receive data as a continuous flow and need to change rapidly and autonomously.

The online system’s ability to adapt to changing data is referred to as its learning rate. A high learning rate means that your system will adapt rapidly to new data but will tend to forget old data quicker while the opposite would apply for a low learning rate making the system less sensitive to noise in new data. This is also a good option in instances where you have limited computing resources allowing you to save space.

Lastly, online learning systems can also be used to train models on datasets that are too large to fit into one machine’s main memory.

Instance based versus Model based learning

Another way to classify a machine learning system is by how the system generalizes. A lot of machine learning tasks involve making predictions. This means that the system needs to generalize to other examples it has not been exposed to. There are two main approaches to generalization; instance and model based learning
